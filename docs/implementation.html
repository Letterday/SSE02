SSE02: Implementation Guide

Implementation patterns

The examples provided in this repository can be used as concrete examples for understanding some of the implementation patterns from Fowler's book on domain-specific languages.

Semantic model

The semantic model is used in all of the examples, specifically you can find it here:
  • Entity-Relation: The semantic model is described by the inner classes EntityType and EntityField, all of which cound be found in the class entityrelation.EntityRelation. The data stored in a concrete entity-relation model is described by an instance of the semantic model and is represented by the inner class Entity
  • QuickGUI: The semantic model is described by the inner classes QComponent, QFrame, QPanel, and QElement, all of which can be found in the class quickgui.QuickGUI. The semantic model is interpreted to provide a concrete instantiation is in the form of a Swing GUI.
  • Serializer: The semantic model is described by the inner classes PacketModel, FieldModel, and its concrete subclasses IntFieldModel, EnumFieldModel, and BooleanFieldModel, all of which can be found in the class serializer.PacketFormat. The model is interpreted by the Packetizer class and stored as raw binary data in the concrete subclasses of the Packet class.
  • Statemachine: The semantic model is described by the three classes of the year2 package: Machine, State, and Transition. The model is directly interpreted to give the behavior of the state machine.
